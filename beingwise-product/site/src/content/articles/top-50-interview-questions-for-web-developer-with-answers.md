## Essential Concepts for Web Developers

Beyond specific coding languages, interviewers often look for a grasp of core web development principles. Understanding these foundational ideas is critical for building robust and user-friendly web applications.

### Q1. What is the difference between progressive enhancement and graceful degradation?

**Ans:** These are two distinct strategies for ensuring web applications work across various browsers and devices. Progressive enhancement starts with a basic, universally accessible version of a website or application, then adds more advanced features and visual enhancements for users with modern browsers. This approach prioritises core functionality for everyone. Graceful degradation, conversely, begins with a fully-featured and visually rich website designed for modern environments. It then ensures that the site gracefully adapts or reduces its functionality and appearance for users with older or less capable technology, maintaining an acceptable user experience even if some features aren't available.

### Q2. Explain CORS (Cross-Origin Resource Sharing).

**Ans:** CORS, or Cross-Origin Resource Sharing, is a browser-based security mechanism that regulates how web pages from one domain can request resources from a different domain. Its primary purpose is to prevent potential security vulnerabilities and data breaches that could occur if web pages had unrestricted access to resources across various origins. CORS works by requiring the server hosting the resources to include specific HTTP headers in its responses. These headers define which domains are permitted to make cross-origin requests and specify the allowed HTTP methods and headers. This policy helps maintain the integrity of web applications while enabling secure, legitimate cross-origin interactions.

### Q3. What is the purpose of the Fetch API in JavaScript?

**Ans:** The Fetch API in JavaScript provides a modern, standardised, and promise-based way to make network requests. Its main purpose is to simplify the process of retrieving data from a server or sending data to a server, replacing older methods like `XMLHttpRequest`. Fetch allows developers to handle HTTP requests asynchronously, which is crucial for building responsive web applications that don't block the user interface while waiting for server responses. It offers more flexibility and power for managing network operations, whether fetching resources like HTML, JSON, or images, or posting data to remote servers.
